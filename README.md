# fgn-scorm-toolkit

Toolkit for authoring and exporting SCORM-compliant courses from FGN sim-based challenges.

This repo is the **stable infrastructure layer** that complements the Lovable-built FGN apps (play.fgn.gg, fgn.academy, broadbandworkforce.com, skill-truck-path). It is not regenerated by Lovable. Code here is hand-written and version-controlled with intent so SCORM compliance, bundle size, and brand integrity stay locked.

## Pipeline at a glance

```
play.fgn.gg challenges      Toolkit                          Distribution
───────────────────         ─────────────────────────────    ──────────────────
Admin publishes             transform()                      Path A: SCORM ZIP
challenges (is_active=true) ↓ reads challenges + tasks       broadbandworkforce.com
↓                           CourseManifest + warnings        skill-truck-path
                            ↓                                external LMSs
                            packageCourse() ────────────► (one path)
                            ↓
                            publishCourse() ────────────► Path B: native rows in
                                                            fgn.academy (courses,
                                                            modules, lessons →
                                                            existing work_orders)

Bridge (zero changes to play.fgn.gg)
─────────────────────────────────────
SCORM Player ──── /mint ────► fgn.academy.scorm-launch-status (new edge function)
              ──── /status ──► poll → join scorm_launch_tokens to user_work_order_completions
                                      (populated by existing sync-challenge-completion
                                      when challenges are completed live)
```

## Standards

- SCORM 1.2 — Phase 1 (universal LMS compatibility)
- cmi5 / xAPI — Phase 4 (richer telemetry, async scoring)
- Both targets generated from the same `course.json` source

## Getting started

```bash
pnpm install
pnpm -r build
```

To export a real challenge as a SCORM ZIP:

```bash
export FGN_PLAY_SUPABASE_URL=https://yrhwzmkenjgiujhofucx.supabase.co
export FGN_PLAY_SUPABASE_ANON_KEY=<from Supabase dashboard>

pnpm --filter @fgn/scorm-builder exec fgn-scorm export <challenge-id> \
  --destination broadbandworkforce \
  --player ./packages/scorm-player/dist/index.html \
  --white-svg ./packages/brand-tokens/assets/logo-fgn-wordmark-white.svg \
  --ink-svg ./packages/brand-tokens/assets/logo-fgn-wordmark-ink.svg \
  --out ./course.zip
```

For the full validation procedure, see [docs/PHASE_1_3_ACCEPTANCE_GATE.md](docs/PHASE_1_3_ACCEPTANCE_GATE.md).

## Repo conventions

- pnpm workspaces
- TypeScript strict everywhere
- Each package has its own `package.json`, `tsconfig.json`, `README.md`
- Brand tokens are the single source of truth — never inline colors, never duplicate the palette
- SCORM compliance is verified via SCORM Cloud + at least one real LMS before any release
- Zero changes to play.fgn.gg — the toolkit is read-only for that platform

## Architecture decisions

| Decision | Rationale |
|---|---|
| Multi-challenge bundling is first-class | Real LMS courses are bundles. Single-challenge ZIPs are a degenerate case. |
| Hybrid versioning | SCORM packages snapshot content at export time; the launch-token bridge syncs completion state live. |
| Reuse fgn.academy work_orders, don't duplicate | fgn.academy already models challenges as `work_orders`. Publisher links lessons to existing rows via `work_order_id`. |
| Existing sync-challenge-completion handles completion | The toolkit consumes the bridge that already exists. Zero new code on play.fgn.gg. |
| SCORM-derived courses are FGN-canonical | No tenant override path. `tenant_id = null` on native publishes. |
| `applyTenantOverrides` in brand-tokens stays | Live multi-tenant FGN surfaces (broadbandworkforce.com under employer branding) still need it. SCORM doesn't. |

### Phase 1.5 scope (queued)

Per architectural clarification on the canonical user journey (Challenge → Work Order → SCORM/Learn), the content model needs alignment:

- New `scorm-launch-status/check` endpoint — read-only completion lookup by `(challenge_id, scorm_student_id)`, no token mint
- SCORM Player on-load gating: locked state when work_order not completed, "create FGN passport" CTA when account doesn't exist, "✓ Work Order Completed" badge when verified
- Drop the `challenge` lesson type from default publisher output — the challenge step belongs to the Work Order layer, not SCORM
- Briefing templates rewritten as post-completion recap framing (not pre-launch briefing)
- Title derivation: module = `[challenge name]: Skills Pathway`; lessons = `[challenge name]: <Step>`
- Reference course.json updated to match new model + re-validation in SCORM Cloud + Moodle

See `docs/PHASE_1_5_CONTENT_MODEL.md` (TBD) for the full plan.

### Phase 1.4 (parallel option)
