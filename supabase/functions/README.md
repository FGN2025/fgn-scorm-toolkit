# Edge functions

Deno edge functions that belong on **fgn.academy's Supabase project** (`vfzjfkcwromssjnlrhoo`). These files are checked in here for review; deployment is via Lovable / the stratify-workforce repo.

## Functions

| Path | Belongs at | Purpose |
|---|---|---|
| `scorm-launch-status/` | `stratify-workforce/supabase/functions/scorm-launch-status/` | Mints SCORM launch tokens and returns completion status. The bridge between SCORM packages and play.fgn.gg challenge completions, with zero changes to play.fgn.gg. |

## How to deploy

For each function:

1. Open the stratify-workforce Lovable project
2. Create a new edge function with the name shown above (e.g. `scorm-launch-status`)
3. Paste in the contents of `index.ts` from this directory
4. Set required env vars (already provided by Supabase: `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`)
5. Deploy via Lovable's commit→build→deploy flow

## Auth

All functions in this directory require an `X-App-Key` header verified against `authorized_apps` via the existing `verify_app_api_key` RPC. Only the `fgn-scorm-toolkit` app slug is accepted for these endpoints.

The plaintext API key is generated by the `provision_fgn_scorm_toolkit_app()` migration function — see `supabase/README.md`.
